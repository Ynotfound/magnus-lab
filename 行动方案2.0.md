## 前言：对"零人类参与"目标的审慎反思

### 比较优势视角

若将 AI 与人类视为经济学意义上的不同生产主体，**分工的原则应是比较优势 (Comparative Advantage) 而非绝对优势 (Absolute Advantage)**。

### 人类 vs AI 的比较优势

| 人类有比较优势的领域 | AI 有比较优势的领域 |
|---------------------|---------------------|
| **战略决策**：决定训练方向、资源分配、优先级 | **执行监控**：7×24 小时不间断指标追踪 |
| **创造性洞察**：发现新的训练技巧、架构创新 | **重复操作**：参数调整、断点重启、日志提取 |
| **价值判断**：什么模型行为可接受/有害 | **模式识别**：从海量日志中发现异常模式 |
| **异常处理**：面对从未见过的故障类型 | **快速响应**：规则明确的故障 (<10 秒恢复) |
| **跨领域迁移**：将其他领域知识应用到训练问题 | **批量处理**：同时管理数十个训练任务 |
| **伦理判断**：训练目标的道德考量 | **精确执行**：无疲劳、无疏忽地执行既定流程 |
| **架构最终决策**：神经网络结构修改批准权 | **架构建议**：基于数据驱动提出修改建议 |
| **可解释性理解**：理解"为什么模型会这样行为" | **特征提取**：自动提取激活模式、电路结构 |
| **幻觉最终裁决**：判断模型输出是否可信 | **幻觉初筛**：自动检测潜在幻觉并标记 |

### 本方案的定位

> **"消除训练过程中所有人类不具比较优势的环节——即重复性、高频率、低创造性的操作——释放人类精力专注于战略决策、创造性洞察与价值判断。"**

**核心纲领（修正版）：** "让 AI 做 AI 擅长的事，让人做人擅长的事"

**设计哲学：** Don't build agents, build skills → **Don't replace humans, empower humans**

## 三、AI 幻觉：原因、检测与缓解

### 3.1 为什么 2026 年 AI 还在幻觉？

> **核心洞见：** 幻觉在理论上可能是不可避免的——类似哥德尔不完备性定理的基本限制，策略应从"消除"转向"管理"。

来源：[Duke University 2026 分析](https://blogs.library.duke.edu/blog/2026/01/05/its-2026-why-are-llms-still-hallucinating/)

### 3.2 幻觉根本原因

| 层级 | 原因 | 说明 |
|------|------|------|
| **数据层** | 训练数据噪声/矛盾 | 训练数据包含错误、矛盾或低质量信息 |
| **架构层** | Next-token 预测机制 | 模型优先预测"流畅"而非"正确"的 token |
| **知识层** | 知识截止/缺失 | 训练数据未覆盖的知识领域 |
| **推理层** | 多步推理误差累积 | 每一步的小错误在长推理链中放大 |
| **解码层** | 采样策略 | 高温采样增加多样性但也增加幻觉风险 |

### 3.3 幻觉类型学

```
┌─────────────────────────────────────────────────────────────┐
│                    AI 幻觉分类                                │
├─────────────────────────────────────────────────────────────┤
│  Intrinsic (内在性)                                          │
│  ├─ 与输入上下文矛盾                                         │
│  └─ 自我矛盾                                                 │
│                                                              │
│  Extrinsic (外在性)                                          │
│  ├─ 无法验证的"事实"                                         │
│  └─ 捏造引用/来源                                            │
│                                                              │
│  Amalgamated (混合性)                                        │
│  ├─ 混合不同来源的正确事实→错误结论                          │
│  └─ 张冠李戴                                                 │
│                                                              │
│  Non-factual (非事实性)                                      │
│  ├─ 逻辑不一致                                               │
│  └─ 物理/数学上不可能的陈述                                  │
└─────────────────────────────────────────────────────────────┘
```

### 3.4 物理小模型的幻觉形式

对于专注物理推理的 8B 小模型，幻觉可能表现为：

| 类型 | 示例 | 风险等级 |
|------|------|---------|
| **公式幻觉** | 捏造不存在的物理公式（如 $F = \frac{Gm_1m_2}{r^3}$） | 🔴 高 |
| **常数幻觉** | 记错物理常数（如 $c = 3\times10^6 m/s$） | 🔴 高 |
| **单位幻觉** | 单位换算错误（如 $1 eV = 1.6\times10^{-16} J$） | 🟡 中 |
| **推导幻觉** | 跳过关键步骤或引入错误中间结论 | 🟡 中 |
| **引用幻觉** | 捏造不存在的论文/定理名称 | 🟠 中 - 高 |
| **概念混淆** | 混淆相似概念（如动量 vs 动能） | 🟡 中 |
| **边界条件幻觉** | 忽略公式适用条件（如相对论效应在低速下） | 🟠 中 - 高 |

### 3.5 小模型特有的幻觉风险

| 风险 | 原因 | 影响 |
|------|------|------|
| **知识压缩损失** | 8B 参数无法完整编码所有物理知识 | 公式/常数记忆模糊 |
| **推理链断裂** | 长推理链中注意力分散 | 多步推导错误累积 |
| **分布外泛化失败** | 训练数据未覆盖的题型 | 完全编造解题方法 |
| **符号操作错误** | 小模型符号推理能力弱 | 代数/微积分错误 |

### 3.6 与蒸馏相关的问题

如果用 Claude 生成 CoT 数据蒸馏到 8B 模型：

| 风险 | 说明 | 缓解措施 |
|------|------|---------|
| **教师幻觉传递** | Claude 的错误推理被学生模型学习 | 专家/规则验证 CoT |
| **过度拟合教师风格** | 学生学会"模仿"而非"理解" | 多样化教师 + 过程奖励 |
| **记忆化陷阱** | 学生记住题目 - 答案对，而非推理方法 | 泛化测试 + 变体题 |

### 3.7 幻觉缓解策略

#### 训练阶段

| 策略 | 具体实现 | 预期效果 |
|------|---------|---------|
| **数据清洗 + 验证** | Claude 生成的 CoT 需经专家/规则验证 | 减少教师幻觉传递 |
| **公式/常数专项强化** | 单独构建公式 - 常数正确性数据集 | 提高基础知识准确率 |
| **符号推理微调** | 用符号计算数据集 (如 SymPy 生成) 微调 | 提高代数/微积分准确性 |
| **Process Reward Model** | 对推理过程每步打分，而非仅最终答案 | 减少中间步骤幻觉 |
| **不确定性训练** | 教导模型在不确定时说"我不知道" | 减少编造行为 |

#### 推理阶段

| 策略 | 具体实现 | 预期效果 |
|------|---------|---------|
| **RAG for Physics** | 检索物理公式/常数手册，而非依赖记忆 | 减少公式/常数幻觉 |
| **Self-Consistency** | 同一问题采样多次，取一致答案 | 减少随机错误 |
| **Step-by-Step Verification** | 每步推理后用工具验证 (如 Wolfram) | 减少推导错误 |
| **Citation Grounding** | 要求每个公式/定理注明出处 | 减少捏造引用 |
| **Unit Check** | 自动检查量纲一致性 | 捕捉单位错误 |

### 3.8 Claude 防幻觉最佳实践

根据 [Anthropic 官方文档](https://platform.claude.com/docs/en/test-and-evaluate/strengthen-guardrails/reduce-hallucinations)：

| 指令 | 效果 |
|------|------|
| **"Allow Claude to say 'I don't know'"** | 明确允许不确定性，减少编造 |
| **"Verify with citations"** | 要求引用来源，强制落地 |
| **"Use direct quotes for factual grounding"** | 用直接引用锚定事实 |

## 四、可解释性监控模块

### 4.1 可解释性工作的意义

**为什么要在训练过程中做可解释性监控？**

| 意义 | 说明 | 实际应用 |
|------|------|---------|
| **提前发现异常行为** | 在模型输出有害内容之前，从激活模式检测异常 | 安全对齐、safety monitoring |
| **理解相变 (Phase Transition)** | 捕捉模型能力突然涌现的时刻 | 决定何时停止训练、保存 checkpoint |
| **区分记忆化 vs 泛化** | 判断模型是真正学会推理还是死记硬背 | 数据质量评估、过拟合检测 |
| **加速调试** | 定位哪一层/哪个模块出现问题 | 架构修改建议的证据支持 |
| **学术贡献** | 可解释性分析本身是可发表的研究 | 论文素材、独立 Credit 项目 |
| **建立信任** | 让人类理解模型为什么做出某个决策 | 高风险场景部署的前提 |

### 4.2 Layer 0: 可解释性监控 Skills

| Skill 名称 | 功能 | 输出 | 负责人 |
|-----------|------|------|--------|
| `magnus_capture_activations()` | 捕获指定层的神经元激活模式 | `ActivationMap[layer_id, neuron_id, step]` | 待组会决定 |
| `magnus_visualize_features()` | 特征可视化 (t-SNE/PCA) | 2D/3D 特征空间图 | 待组会决定 |
| `magnus_detect_phase_transition()` | 检测训练相变 (能力涌现点) | `PhaseTransitionEvent[step, metric_jump]` | 待组会决定 |
| `magnus_analyze_memorization()` | 区分记忆化 vs 泛化 | `MemorizationScore[0-1]` | 待组会决定 |
| `magnus_trace_circuit()` | 追踪特定功能对应的电路 | `CircuitGraph[nodes, edges, weights]` | 待组会决定 |
| `magnus_ghost_gradient()` | Ghost Gradients 激活 dormant neurons | `ActivationReport` | 待组会决定 |

### 4.3 可解释性与幻觉检测的结合

| 可解释性技术 | 幻觉检测应用 |
|-------------|-------------|
| **激活模式分析** | 某些概念对应神经元未激活→可能编造 |
| **相变点分析** | 能力涌现前后幻觉率对比 |
| **记忆化检测** | 区分真正理解 vs 死记硬背 |
| **电路追踪** | 定位"推理 circuit"vs"编造 circuit" |

### 4.4 可解释性工作流

```
训练过程
    ↓
[激活捕获] → 每 N 步捕获一次关键层激活模式
    ↓
[特征分析] → t-SNE/PCA 降维可视化
    ↓
[相变检测] → 发现能力涌现点 → 自动保存 checkpoint
    ↓
[记忆化分析] → 区分真假学习 → 数据建议
    ↓
[电路追踪] → 定位功能对应的 neural circuit
    ↓
[人类解读] → 研究员分析可视化结果 → 架构/数据决策
```
